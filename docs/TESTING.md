# Testing

## 목적

이 문서는 AI 에이전트가 코드 구현 후 검증하는 기준과 절차를 정의합니다.

## 검증 원칙

각 task 구현 후 아래 순서로 검증합니다.

1. 관련 코드 수정 완료 여부 확인
2. 현재 실행 확인 모드에 맞는 정적 점검 또는 로컬 실행 확인
3. `docs/STYLEGUIDE.md` 기준으로 JavaScript/MCP tool 규칙 위반 여부 확인
4. 현재 task의 구현 범위와 완료 조건 일치 여부 확인
5. 진행 현황 문서 갱신

## 실행 확인 모드

현재 저장소의 기본 실행 확인 모드는 `수동 실행 확인`입니다. AI가 직접 실행 확인까지 수행하면 토큰 소모가 크기 때문에 기본은 사람이 실행과 결과 확인을 담당합니다.

- `수동 실행 확인`: 실제 MCP 클라이언트(Claude Desktop, Claude Code 등) 연결 확인처럼 사람이 직접 해야 하는 확인이 필요할 때 사용합니다.
- `에이전트 확인`: 에이전트가 MCP inspector 또는 로컬 테스트 스크립트로 tool 목록 조회와 호출 결과를 직접 확인합니다.

## 수동 실행 확인 모드

- 사람은 실제 MCP 클라이언트에 서버를 연결해 tool 목록 노출과 실제 호출 결과를 확인합니다.
- 이 모드는 클라이언트 통합 확인이 필요할 때, 또는 사용자가 요청할 때 사용합니다.
- 실행 명령은 현재 저장소의 기존 실행 방식과 빌드 구조를 우선 사용합니다.

## 에이전트 확인 모드

- 에이전트는 코드 눈검토, 관련 파일 점검, MCP inspector 또는 테스트 스크립트를 통한 tool 목록·호출 확인까지 수행합니다.
- 정상 입력과 잘못된 입력 모두에 대해 응답과 에러 처리를 확인합니다.
- 아직 연결되지 않은 다음 단계 tool을 현재 task 확인 항목에 섞지 않습니다.
- 실패 시 어떤 tool, 어떤 입력, 어떤 응답 또는 에러가 문제인지 기록할 수 있어야 합니다.
