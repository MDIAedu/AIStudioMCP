# Style Guide

## 목적
이 문서는 AI 에이전트가 코드를 작성할 때 따라야 할 JavaScript, MCP tool 정의, 주석 규칙의 기준입니다.

## 공통 원칙

- 처음 보는 사람도 읽을 수 있도록 과도한 난이도의 패턴은 피합니다.

## JavaScript 규칙

- 함수와 변수 이름은 역할이 드러나게 작성합니다.
- 함수의 입력값은 사용 전에 형태를 확인하고, 예상과 다른 값이 들어오면 조용히 넘어가지 않고 명확하게 실패시킵니다.
- 지금 task에 필요한 동작만 작성하고, 미래 확장을 가정한 추상화는 기본적으로 추가하지 않습니다.

## MCP Tool 규칙

- tool 이름은 동사형으로 명확하게 작성합니다.
- tool의 `description`은 이 tool을 호출하는 주체가 사람이 아니라 LLM이라는 점을 감안해, 언제 이 tool을 써야 하는지와 입력/출력 의미를 LLM이 오해 없이 판단할 수 있도록 구체적으로 작성합니다.
- 입력 스키마는 `zod` 등으로 명시적으로 검증하고, 각 파라미터 설명도 tool description과 같은 기준으로 작성합니다.
- 하나의 tool은 하나의 명확한 동작만 담당합니다. 여러 개의 관련 없는 기능을 하나의 tool에 몰아넣지 않습니다.
- 에러 응답은 실패 사실만 알리지 않고, LLM이 다음 행동을 판단할 수 있도록 구체적인 원인을 담아 반환합니다.

## 외부 자산/설정 규칙

- API 키, 인증 토큰 등 민감한 값은 코드에 직접 적지 않고 환경 변수 또는 별도 설정 파일로 분리합니다.
- 임시로 사용한 값과 실제 사용할 값은 구분 가능하게 관리합니다.

## 주석 규칙

- 초보자 기준으로 이해를 돕는 주석은 허용합니다.
- 주석은 기본적으로 한글로 작성합니다.
- 왜 이런 분기나 상태 처리가 필요한지처럼 코드만으로 바로 드러나지 않는 맥락을 우선 기록합니다.
- 함수에는 어떤 기능을 하는 함수인지 꼭 주석을 작성합니다.

## 금지 또는 제한 규칙

- 현재 task와 무관한 패키지, 모듈, 시스템 도입은 하지 않습니다.
- 민감한 계정 정보, 키, 외부 서비스 비밀값을 코드, 문서, 로그에 적지 않습니다.
- "나중에 쓸 수도 있다"는 이유만으로 미사용 상태, 더미 데이터 구조, 가짜 설정 항목을 추가하지 않습니다.
